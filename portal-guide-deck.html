The Ascent Portal

The web app that runs Foundations end to end — sign-up, learning, scoring, and career direction. What it is, what's in it, and how it's used.

FOR STAKEHOLDERSPLAIN-LANGUAGE OVERVIEW
IN ONE IDEA

One web app takes a student from sign-up → through 8 sessions of real work → to a tech career direction — and gives staff the tools to run it.

Two sides: a personal page for each student, and secure dashboards for facilitators and the program manager.

Sign-up

✅

Readiness check

Confirms a computer + reliable internet. No? Flagged "needs help" so staff resolve access before Session 1.

📝

Enrolment

Name, contact — plus a guardian for under-18s. Minimal data only.

🔏

Consents

Explicit opt-ins for data, messaging, and guardian consent — all recorded.

Onboarding & the warm-up (Day 0)

📨

3 setup missions

Power on → get online + create a free Google account → join a practice video call. Sent before Session 1.

⏱️

10-minute warm-up

Four quick tasks (mouse, typing, find-a-file, search). Not a test, not graded.

Outcome: each student is tagged Bridge or Foundation + a baseline typing speed — used only for pairing & pacing, never shown as a ranking.
THE HEART OF IT

The Learner Portal — one personal link

📚

8 session cards

Live session, walkthrough & sandbox links, plus the assignment.

📤

Submit work

Paste a Google Drive link; resubmit anytime to improve.

📈

Mastery profile

Best level per skill — only ever goes up.

💬

Feedback

Facilitator notes appear on each submission.

✍️

AI-use policy

Read & sign once; applies to the capstone.

🧭

Compass

Unlocks their free career direction.

Facilitator dashboard · password-protected

👥

Roster

Every learner: cohort, tag, typing speed, progress (e.g. 5/8), consents, and a link to their portal. A "needs help" alert flags access issues.

🎯

Scoring & feedback

Per submission, pick a skill + level (0–3) + a note. Best-evidence, never averaged. Set a learner's pace (extension / support).

🔗

Cohort resources

Paste each session's live / walkthrough / sandbox links — they appear on every learner's portal.

📊

Compass insight

Completions, debrief requests, and which Tracks are in demand.

Admin dashboard · password-protected

📊

Headline stats

Enrolments, enrolled vs. needs-help, cohorts, submissions, Compass completions.

🗂️

Cohorts

Create, rename, set start date, open/close enrolment, delete. Can't delete a cohort with learners (safety check).

👤

Enrolments

Change status, move between cohorts, or remove a learner.

Compass — where it leads

After Foundations, Compass reads what the student actually did — their skills profile — plus a short interest check, and recommends a tech path in plain language, using their own work as evidence.

A primary path

Plus one to explore — Data, Web/Software, AI Ops, or Design.

A real conversation

They can request a one-to-one debrief.

On to Tracks

The chosen path's skill course comes next.

Access, privacy & status

🔐

Access & privacy

  • Students: a private personal link — no account to manage
  • Staff areas: password-protected, not search-indexed
  • Minimal data + recorded consents; minors need guardian consent
🛠️

Live vs. coming next

  • Live: sign-up, onboarding + diagnostic, learner portal, scoring, admin, Compass
  • Planned: auto-sending onboarding messages; serving teaching decks in-app
