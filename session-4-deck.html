  • 1Sort — sort the whole table, never one column. (Sort one column alone → rows shred. Undo immediately.)

$A$1 vs A1 — the rule that breaks everything if you skip it

Tax rate 0.08 lives in $B$1. We fill a tax column down. One of these survives the fill-down. One doesn't.

CellWRONG — relative B1What breaks on fill-downRIGHT — $B$1
E4=D4*B1B1 slides to B5, B6… — now pointing at prices, not the tax rate. Tax = subtotal × a random price, or 0 on a blank cell.=D4*$B$1
E5=D5*B2 (slid)Multiplies the subtotal by an item price — a plausible-looking wrong number.=D5*$B$1 (stays put)
The rule: $ before the letter locks the column, before the number locks the row, $B$1 locks both. If a cell should mean the same thing in every row, lock it. (Press F4 to cycle the $.)

Forms — a validated front door for your data

Build the form

  • Short text — item name (required)
  • Number — amount · validate > 0
  • Date — picker, so it's a real date
  • Dropdown — category, so it's never free-typed three ways
  • Required toggle on the fields that matter

Feed the sheet

  • Responses tab → Link to Sheets
  • Every submission = one new clean row
  • Validation already typed it: numbers are numbers, dates are dates
  • Your SUM / AVERAGE / chart update automatically
Validation is rules at the door. Clean data in → trustworthy machine out. The form turns data collection into data.

AI gives you a formula — you verify it

🚩 Inspect first

  • Real formula? =SUM(B2:B16) ✔
  • Or hardcoded? =4500 ✘ — doesn't recompute
  • Hardcoded values are penalized even when the AI suggested them

✅ Then hand-verify

  • Cell shows 4,820
  • Re-add 3 rows by hand: 300+150+275 = 725
  • Re-add the full column → 4,820. Match = trust.
  • Mismatch = wrong range or a text-not-number cell
The rule: trust is earned by verification, not by the source. Leave a one-line note: "Re-added rows 2–4 (725) and full column (4,820) — matches =SUM(B2:B16)."

Think Like a Builder

1A formula is an algorithm

A precise recipe a machine runs the same way every time. You didn't use a calculator today — you built one.

2Decompose the calculation

Break it into ordered steps: price × qty = subtotal, then subtotal × tax = tax owed. One step per rule.

3References are rules

$B$1 vs B1 is a rule about what stays fixed and what moves as the machine runs.

4If it breaks, debug it

The fill-down failure wasn't bad luck — find the wrong rule, explain it, fix it. That's debugging.

This thread runs to Session 8 and is measured by the new PROBLEM-SOLVING competency. Decompose · rule · debug — for the rest of the course.
