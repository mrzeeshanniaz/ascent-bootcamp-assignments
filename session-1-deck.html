THE HEART OF THE SESSION ♥

Recovery Lab — "gone" almost never means gone

🗑️

1Trash

Like "Recently Deleted." Delete → Trash → Restore. Drive keeps it 30 days.

⏱️

2Version history

The time machine. Overwrote your work? Scroll back and restore an earlier version.

👯

3Conflict copy

Two near-identical files. Compare, keep the superset, archive the loser — never delete blind.

After today: you never panic about losing work again.

Version history — the magic, step by step (Google Docs)

  • 1Name the good version first: File → Version history → Name current version → good copy
  • 2Destroy it on purpose: Ctrl/Cmd + A, type garbage. It auto-saves — "I never pressed save."
  • 3Travel back: File → Version history → See version history (Ctrl/Cmd + Alt + Shift + H)
  • 4Click good copy → Restore this version → it's back. ✨
Nothing is lost by looking. Restoring just adds a new version on top — you can't make it worse by exploring.

Assignment 1 — in your own Google Drive

Build & recover

  • Nested directory: ≥ 3 levels, ≥ 6 files, named YYYY-MM-topic-vN
  • Overwrite a file → recover via version history
  • Delete a file → restore from trash
  • Submit: screenshot of the tree + a 3-sentence recovery note

The grading bar (no secrets)

  • Proficient: logical nesting + consistent naming; both recoveries correct
  • Mastery: naming shows a system; note explains when to use version history vs. trash
